1. Overview
The SnippetShelf Backend API provides RESTful endpoints for managing and retrieving data related to code snippets, collections, team activities, and analytics. It serves as the data layer for the SnippetShelf frontend application.
All responses are in JSON format, and timestamps use ISO 8601 format (e.g., 2023-10-27T10:00:00Z).

2. Configuration & Setup
2.1 CORS Configuration
Your Spring Boot backend must be configured to allow requests from the React frontend (http://localhost:3000).
Example configuration using WebMvcConfigurer:
@Bean
public WebMvcConfigurer corsConfigurer() {
    return new WebMvcConfigurer() {
        @Override
        public void addCorsMappings(CorsRegistry registry) {
            registry.addMapping("/api/**")
                    .allowedOrigins("http://localhost:3000")
                    .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                    .allowedHeaders("*")
                    .allowCredentials(true);
        }
    };
}

5. Example Integration (Frontend)
Fetch Dashboard Stats (React Example):
async function getDashboardStats() {
  const res = await fetch('http://localhost:8080/api/dashboard/stats');
  if (!res.ok) throw new Error('Failed to fetch stats');
  return await res.json();
}
